📌 How subscriptions work in the PHP rebuild: Each organization registers as a tenant. A subscriptions table tracks plan_type, start_date, end_date, and status. A PHP middleware checks expiry on every request — if expired, only the dashboard and billing page are accessible. Features are gated by plan level stored in session. Payment can be manual (GCash/bank transfer + admin confirms) or later integrated with PayMongo/Stripe.

✅ YES — 100% Replicable in PHP

Everything in this Spring Boot project can be done in plain PHP + MySQL. The Spring framework provides convenience (ORM, security, templating) but nothing it does is impossible in PHP. The logic — CRUD operations, file uploads, session auth, PDF generation — is entirely achievable.

⚠️ Hardest Part: PDF Reports

The 10 JasperReport templates (.jasper files) are compiled binary Java templates. They cannot run in PHP. You must rebuild each report using TCPDF or mPDF. This is the biggest time investment. Reuse the existing layout/design but rewrite the rendering logic.

💡 Big Win: Reuse HRIS v1 Frontend

Since your HRIS v1 already uses HTML/CSS/JS, you can copy the UI components, sidebar, navbar, tables, and modals directly. The Thymeleaf templates are plain HTML with just th: attributes — strip those and replace with PHP <?php echo ?>.

🆕 New: Multi-Tenant + Subscriptions

The Java system is single-tenant (one organization). The PHP rebuild should add organization_id to every table to support multiple clients subscribing. Each org is isolated — HR admins only see their own employees. Plans gate features by checking the active subscription.
